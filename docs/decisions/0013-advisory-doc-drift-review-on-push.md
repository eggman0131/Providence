# 0013 — Advisory doc-drift review on every push

- **Status:** Accepted
- **Date:** 2026-07-02
- **Deciders:** Founding session (human director + agent)
- **Related:** [`../30-ai-agent-contract.md`](../30-ai-agent-contract.md) (I6, §6, §9), [`0002`](./0002-llm-as-strategic-advisor.md), [`0005`](./0005-macbook-only-offline-runtime.md), [`0010`](./0010-branch-workflow-and-ci.md), [`0011`](./0011-advisory-code-scanning.md), [`0012`](./0012-project-management-on-github.md)

## Context

**I6 (docs-as-code)** demands that "docs and code must never drift; a stale doc is a defect." But the contract's own governing principle (§6.1) is that *anything not enforced by a tool is only a recommendation* — and doc↔code drift is, so far, **unenforced**. The gate mechanises I1–I5 and I8–I9; I6 has no check.

I6 cannot be made a hard, deterministic gate check the way boundaries or magic numbers are: deciding whether a doc has *genuinely* gone stale is a judgement, not a lexical fact. Two existing precedents show the way to enforce it anyway, without compromising the gate:

- **[ADR 0002](./0002-llm-as-strategic-advisor.md)** — a non-deterministic LLM may *advise* a deterministic engine that *actuates*. Apply the same split to governance: an LLM judges drift; deterministic code decides candidates and files the issue.
- **[ADR 0011](./0011-advisory-code-scanning.md)** — advisory *reporting* that never gates is compatible with I9. The gate stays the sole arbiter of "green"; this adds breadth of reporting, not a new authority.

The LLM must be **local** (I7: offline-capable, MacBook-only). A GitHub Actions runner cannot run the local model, so — unlike CodeQL in ADR 0011 — this cannot live in CI; it runs on the developer machine, exactly like the existing local, ollama-driven graphify post-commit hook.

## Decision

We will add an **advisory doc-drift review that runs on every push**, built as a deterministic detector, a local-LLM advisor, and a deterministic actuator, and wired to a **non-blocking `pre-push` git hook**. It is **never a gate**.

1. **Deterministic detector — `cargo xtask doc-review [--since <ref>] [--json]`.** A new, pure xtask subcommand (`xtask/src/doc_review.rs`) that, for a git range, reports **which** governed docs a change may have staled and **why**, via three signals: (a) a changed path falls under an area a doc owns (the `doc-review.toml` map at the repo root); (b) a doc cites an ADR whose file changed; (c) an I6 co-change catch-all — code changed but the range updated no doc at all. A doc edited in the same range is never a candidate. It shells out to `git` read-only, adds no dependency, and is **not** part of `cargo gate`.
2. **LLM advisor — a pluggable agent, local by default.** The orchestrator asks an agent to judge each candidate: is the drift real, and is the **doc** wrong (update the doc) or the **code** wrong (the doc mandates behaviour the change violates)? The review contract and procedure are committed as Cline primitives — a rule (`.clinerules/doc-drift-review.md`) and a workflow (`.clinerules/workflows/doc-drift-review.md`) — and the same prompt drives whichever agent runs. Two agents are supported (`--agent`): the **default is local Qwen via Cline** (`qwen3.6:35b-mlx`, offline, I7); an **online Claude Haiku** agent (via the `claude` CLI) is the escalation. Either way the advisor has **no side effects** — it only emits a JSON verdict.
3. **Inconclusive ≠ clean; escalate to an online agent.** A local model sometimes returns no parseable verdict (format non-compliance, a Cline/ollama hiccup, or a missing tool). That is a **missed** review, not a "no drift" pass, and the orchestrator says so **loudly**, listing the unreviewed docs and the exact re-run command: `scripts/doc-review/run.sh --agent claude …`. Haiku is markedly more reliable at the strict JSON contract, so it is the recommended escalation. The online agent is an **explicit, dev-time** call (like `gh` here, or `cargo deny fetch`): it is never automatic and never part of the game runtime, so **I7 holds** — the offline-capable local agent remains the default and the game ships no cloud dependency.
4. **Deterministic actuator — issues, deduplicated.** `scripts/doc-review/run.sh` (+ `post_issues.py`) runs the detector, invokes the chosen agent, and for each verdict of real drift at ≥ medium confidence opens a GitHub **Issue** (per [ADR 0012](./0012-project-management-on-github.md)) via `gh`, labelled `doc-drift` + `area:docs` (+ `adr-needed` when an architectural doc must change). Issues carry a hidden fingerprint (the **doc path**); the orchestrator scans open `doc-drift` issue bodies and **skips duplicates**, so re-pushing does not spam and the model's run-to-run "which side" wobble cannot file two issues for one drift (the body still records the side). The agent never writes to the tracker — dedup, labels, and creation are deterministic (the ADR 0002 split).
5. **Non-blocking, on push.** Git has no `post-push` hook; `pre-push` is the only client push event. The hook (`scripts/hooks/pre-push`, installed as a thin shim by `cargo xtask setup`) launches the review **detached** and exits 0 immediately, so `git push` is never delayed or aborted. `PROVIDENCE_SKIP_DOC_REVIEW=1` disables it. Background log: `~/.cache/providence-doc-review.log`.
6. **Non-gating, always.** Like CodeQL (ADR 0011), this must **never** become a required check and does **not** change `cargo gate`. Mergeability remains defined solely by the gate (I9 stays literally true — the review is reporting, and the only network touches are `gh` to the tracker and, on an explicit escalation, the online agent — both dev-time actions, not game/runtime dependencies).

## Consequences

- **Positive:** I6 gains a mechanical (if advisory) check at last; drift surfaces as a tracked, deduplicated Issue that says which side to fix; the judgement runs fully local and offline by default (I7); reuses the proven graphify hook pattern and the existing issue tracker (ADR 0012); no new Rust dependency, no boundary change, and the gate is untouched. An inconclusive local review is surfaced **loudly** with a one-line online-agent re-run command, so a flaky local model degrades to "please re-review," never to a false "no drift."
- **Negative / trade-offs:** depends on tooling being present and configured (ollama + the `qwen3.6:35b-mlx` model, the Cline CLI pointed at it, an authenticated `gh`, and — for the escalation — the `claude` CLI) — when the core tools are absent the hook logs and skips; the model's verdicts are non-deterministic and may miss or over-flag (mitigated by the ≥ medium-confidence bar, the inconclusive-escalation path, human triage, and dedup); the online escalation spends **paid API tokens per run** (local Qwen is free, owned compute) and its latency is **network-bound and variable** — which is exactly why it is a manual, operator-invoked step and not the default, and it never runs on the game runtime; its role is the **fallback when the local model returns no usable verdict** — not that it judges better (both caught the validated drift equally, see below); `pre-push` fires just before the remote accepts refs, so a *rejected* push still triggered a review of local commits (harmless — the subject is local docs/code and the fingerprint prevents double-filing on the later successful push); a new root file (`doc-review.toml`) and a `pre-push` hook now exist.
- **Validation (empirical, small sample):** exercised against a deliberately introduced drift — dropping `#![no_std]` from the core, which `docs/20-architecture.md` mandates. The deterministic detector flagged the right docs, and **both** the local **Qwen** default and the online **Haiku** escalation caught it **3/3 at high confidence**, differing only on the doc-vs-code *side*, never on whether drift existed. With the diff inlined into the prompt the local review takes ~40–90 s (no agentic file-wandering — the earlier multi-minute runs were the un-hardened path). Caveat: a *judgement-dependent* drift (a bespoke CI step vs "CI runs only the gate") was caught only intermittently, and a confident false "no drift" is indistinguishable from a genuine clean pass and does **not** escalate — so soft/interpretive drift is where a miss can hide. This is a helpful net, not a guarantee, which is why it stays advisory.
- **Enforcement / gate impact:** **none.** No new gate check; `cargo gate` is unchanged and remains the single definition of "green." Branch protection's required check stays `gate` only — adding this to required checks would violate this ADR and I9. `cargo xtask setup` additionally installs the pre-push shim (best-effort, never fails provisioning). `doc-review.toml` lives at the repo root (like `deny.toml`/`rustfmt.toml`), outside `config/`, so the config-validity and config-key checks do not see it. Editing the map is a tooling change, not an ADR-level change.
- **Docs to update:** this ADR, `decisions/README.md` (index), and `CLAUDE.md` (a "Document drift review" section documenting the hook, prerequisites, escape hatch, manual command, and log path; the `doc-drift` label under Project management). A `doc-drift` GitHub label is created out-of-band.

## Alternatives considered

- **Run it in GitHub Actions on `push`.** Matches ADR 0011's CI home, but a cloud runner cannot run the *local* model, and making a cloud LLM the **default** review path would break offline-by-default (I7) — rejected as the default. (An online agent is still available as an *explicit* escalation, which is different from making CI the default.)
- **Auto-escalate to the online agent on every local failure.** Convenient, but it would turn a routine push into an implicit, recurring cloud call (cost + network) without the operator asking — eroding offline-by-default. Rejected in favour of a loud message + a one-line manual `--agent claude` re-run (an opt-in env default is left available for those who want it).
- **Make it a hard gate check.** Would give I6 the same teeth as I1–I5, but the judgement is non-deterministic and needs network + a heavy local model, which would wreck the gate's determinism, speed, and offline reproducibility (I3/I9). Rejected; advisory is the correct strength.
- **Call ollama directly instead of Cline.** Simpler and matches the graphify hook, but Cline gives reusable, committed **rules/workflow** primitives (and room to grow: MCP, skills) and is the local agent already in use here. Chosen for the advisor; the model underneath is still local Qwen either way.
- **Let the LLM open issues itself** (Cline actuates via `gh`). More "Cline-native," but surrenders deterministic control of dedup, labels, and fingerprints and gives the model unsupervised write access to the tracker. Rejected in favour of the ADR 0002 advisor/actuator split.
- **`post-commit` instead of `pre-push`.** Would reuse the graphify trigger, but reviews every WIP commit rather than what is actually shared, and the ask was explicitly "on push." Rejected.
